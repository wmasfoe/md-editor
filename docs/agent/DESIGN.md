# 设计规范

## 事实来源

- 状态：草案
- 最后更新：2026-06-19
- 主要产品界面：macOS 桌面外壳、文件树、大纲、所见即所得编辑器、源码编辑器、资源预览和原生菜单。
- 已审阅证据：`README.md`、`docs/agent/markdown_editor_requirements.md`、`docs/agent/v0_1_polish_improvements.md`、`apps/desktop/src/app/App.tsx`、`apps/desktop/src/app/useDesktopEditorController.ts`、`apps/desktop/src/components/FileTreePanel.tsx`、`apps/desktop/src/styles.css`、`apps/desktop/src-tauri/src/lib.rs`，以及 `packages/editor-ui/src/components/` 下的编辑器组件。
- 证据边界：本次审视基于仓库中的行为实现与样式。仓库内没有用户研究记录、产品分析数据或持续维护的截图基线。

## 品牌

- 个性：克制、准确、本地优先、值得信赖；它首先是技术作者的专注写作工具，而不是缩小版 IDE。
- 信任信号：清晰的文件身份、明确的已保存/未保存状态、Markdown/MDX 无损处理、可预测的键盘行为和可撤销操作。
- 避免：常驻的密集工具栏、IDE 式面板堆叠、隐藏状态、无解释的模式变化、中英文混用，以及写作过程中的装饰性动效。

## 产品目标

- 目标：让打开、理解、编辑、导航和保存 Markdown/MDX 文档成为连续体验；不让语法干扰写作，也不隐藏文档真实内容；让新用户能够发现熟练用户使用的快捷能力。
- 非目标：首个设计周期内不做知识图谱、默认双栏预览、项目管理或完整代码 IDE。
- 成功信号：首次使用者不依赖系统菜单即可打开文件夹并开始写作；用户始终知道当前文件、编辑模式和保存状态；常用格式不要求背诵 Markdown；切换视图时不会感到上下文丢失。

## 用户与任务

- 主要用户：技术写作者、文档维护者、博客作者和 MDX 内容创作者。
- 用户任务：打开已有笔记或项目；快速撰写和格式化正文；插入代码、链接、列表、表格和图片；在长文档中导航；必要时检查精确源码；确认内容已安全保存。
- 使用场景：macOS 上长时间以键盘为主的写作、中英文混排、带资源目录的文档项目，以及偶尔对 Markdown/MDX 进行源码级修复。

## 信息架构

- 主导航：可收起的工作区侧栏包含“文件”和“大纲”两个明确入口，文档画布始终是稳定的主界面。
- 核心界面：欢迎/空状态、文档编辑器、源码编辑器、资源预览和可恢复的错误状态。
- 内容层级：窗口/标题信息 → 紧凑文档栏 → 写作画布 → 上下文格式与块操作 → 可选侧栏 → 低强调度状态信息。
- 文档栏应包含侧栏开关、文件名、未保存标记、模式切换和更多操作，但不应演变成大型功能区。
- 资源预览属于临时文档视图，必须提供明确的“返回文档”操作，不能依赖下一次编辑隐式关闭。

## 设计原则

- 状态先于命令：先展示文件、模式、保存、加载和错误状态，再增加更多操作。
- 写作默认优先：控件在不相关时保持安静，但核心状态和恢复操作必须始终可见。
- 渐进披露：高频格式操作靠近选区或光标，低频操作进入菜单或命令面板。
- 保持连续：切换编辑模式或资源视图时，保留光标、选区、滚动位置、侧栏状态和文档身份。
- 键盘优先但不只支持键盘：每个快捷键操作都需要可发现的菜单、标签、提示或命令面板入口。
- 权衡：使用小型常驻文档栏和上下文控件，而不是大型常驻工具栏；在具备可靠恢复机制前，以明确保存状态代替完全不可见的自动保存。

## 当前交互审视

### 已观察事实

- `App.tsx` 渲染了固定 272 px 侧栏和编辑器/资源界面，但没有文档标题栏或状态区，因此主窗口内看不到文件身份、未保存状态和当前编辑模式。
- “文件”和“大纲”是互斥的侧栏标签。React 布局中的侧栏始终存在，当前快捷键只是切换侧栏内容，并非真正隐藏侧栏。
- 未打开文件夹时只提供“打开文件夹”按钮。默认未命名文档已经填入说明性 Markdown，因此首屏不是有意设计的欢迎体验。
- 大部分文件和模式命令位于英文原生菜单中，可见侧栏与操作进度使用中文，未保存确认和后端错误又使用英文。
- 所见即所得与源码编辑器共享居中的 860 px 内容宽度，但源码模式改变了字体、行号区和当前行样式，画布内却没有模式标识。
- 源码模式下按 `Command+F` 会直接显示 CodeMirror 默认搜索面板，没有产品级主题和布局处理。其旧式、近似浏览器原生控件的视觉表现与当前克制的桌面界面不一致，使高频搜索功能显得尚未完成。
- 常用格式主要依赖 Milkdown 快捷键。`markdown-format-commands.ts` 中注册的格式命令仍是占位实现，尚未形成可用的可视化命令入口。
- 打开图片后，编辑器会被资源预览替换。返回方式是隐式的：下一次编辑会清除资源状态，但预览界面没有明确的返回操作。
- 删除与放弃修改使用浏览器 `window.confirm`，产品无法控制信息层级、文案，也无法提供“保存后继续”。
- 错误以顶部横幅持续显示，进行中操作使用悬浮提示；由于缺少保存状态界面，保存成功没有稳定、可见的状态变化。

### 设计推断

- 主要能力隐藏在快捷键和 macOS 菜单中，导致应用看起来比实际能力更弱。
- 缺少文档身份和保存状态，会在长时间写作中持续制造不确定感，即使底层数据处理本身正确。
- 在最小 760 px 窗口中，永久显示且不可收起的侧栏会挤压画布，削弱低干扰写作体验。
- 模式切换缺少稳定控件和可见确认，光标/滚动连续性也尚未成为明确的产品契约，因此切换过程容易显得突兀。
- 当前体验更重视渲染保真和文件操作，而不是即时写作过程；选区格式、块插入、链接编辑、空段落和恢复操作都缺少提示。

### 优先改进项

1. P0，建立方向感与信任：增加紧凑文档栏，展示当前文件名、未保存状态、所见即所得/源码分段切换、侧栏开关和更多菜单。原生窗口标题同步为“文件名 - Markdown Editor”，未保存时增加明确标记。
2. P0，明确首次操作：用真正的欢迎页替换预填充的未命名文档，提供“新建文档”“打开文件”“打开文件夹”和最近文件。进入编辑后使用低强调度首行占位提示，避免示例内容被误保存。
3. P0，让侧栏可选：支持显示/隐藏，记忆宽度和最后使用的标签，在窄窗口中自动收起。“文件/大纲切换”和“侧栏显示/隐藏”必须是两个独立概念。
4. P0，保护写作连续性：按模式保存光标/选区和滚动位置；保存及非破坏性命令结束后恢复编辑器焦点；资源预览增加明确返回操作；无关状态变化不得重新挂载编辑器。
5. P1，增加上下文写作控件：选中文本时展示小型浮动工具栏，支持粗体、斜体、行内代码、链接和删除线；通过 `/` 菜单或块手柄插入标题、列表、引用、代码块、表格、图片和 MDX 组件。不要增加永久密集工具栏。
6. P1，统一反馈：同一语言环境只使用一种产品语言；用无障碍对话框替代浏览器确认框，提供“保存”“不保存”“取消”；只有操作确实可感知时才显示保存进度，随后稳定显示“已保存”或“未保存”。
7. P1，重做源码搜索体验：使用应用现有设计变量和紧凑桌面控件重新设计 `Command+F` 搜索面板，同时保留 CodeMirror 的搜索行为和键盘语义。统一提供搜索/替换、匹配数量、上一个/下一个、区分大小写、全词匹配、正则表达式和关闭操作；随后补充拼写检查策略、链接编辑、拖拽插入提示、图片替代文本编辑、代码块键盘操作以及可从文档栏进入的命令面板和快捷键帮助。
8. P2，增加专注写作选项：打字机滚动、专注模式、侧栏自动隐藏、可读宽度预设，以及浅色/深色/跟随系统主题。这些是个性化设置，不应早于核心流程修复。

## 视觉语言

- 颜色：保留安静的中性画布和克制的青蓝强调色，将硬编码灰色与语义反馈色收敛为设计变量。未保存、错误和选中状态不能只依赖颜色表达。
- 字体：界面使用清晰易读的无衬线字体；所见即所得模式使用适合中英文混排的正文栈；源码与代码使用稳定等宽字体。后续允许用户调整正文字号和行高。
- 间距与布局：以 4 px 为基础网格；紧凑控件高 28-32 px；文档栏高 40-44 px；大窗口画布边缘留白 64-80 px；正文可读宽度可在 680-860 px 范围配置。
- 形状与层级：桌面控件使用小圆角和低层级阴影；阴影只用于临时菜单、对话框和选区工具。
- 动效：披露与状态变化使用 120-180 ms；保存时画布不能位移；遵守系统减少动态效果设置。
- 图标：统一使用 16 px 线性图标，并配合文本标签或提示；不常见操作不能只靠图标解释。

## 组件

- 复用现有组件：`FileTreePanel`、`OutlinePanel`、`MilkdownEditor`、`SourceEditor`、`AssetPreview`、现有侧栏标签、代码块控件和主题变量。
- 新增或调整组件：`DocumentBar`、`WelcomeState`、`SaveStateIndicator`、`EditorModeSwitch`、`SidebarToggle`、`SelectionToolbar`、`BlockInsertMenu`、主题化的 `SourceSearchPanel`、`CommandPalette`、`ConfirmDiscardDialog`、`ToastRegion` 和明确的 `AssetPreviewHeader`。
- 变体与状态：文档栏包含已保存/未保存/保存中/错误；侧栏包含展开/收起/窄屏；编辑器包含所见即所得/源码/只读；工具栏包含隐藏/选区/链接；对话框包含保存/放弃/删除；欢迎页包含有/无最近文件。
- 所有权：语义设计变量归桌面主题 CSS 管理；编辑内容变量保留在可复用的 `editor-ui`；命令 ID 和快捷键继续由 `editor-core` 管理；可见控件与原生菜单必须分发同一组命令 ID。

## 无障碍

- 目标标准：Web 渲染界面达到 WCAG 2.2 AA；由 Tauri 管理的控件优先采用平台原生语义。
- 键盘与焦点：提供完整 Tab 顺序、清晰焦点、标签/菜单方向键操作、Escape 关闭临时界面和焦点恢复，不能存在仅悬停可用的命令。源码搜索中，`Command+F` 应聚焦并选中搜索词，Enter/Shift+Enter 切换匹配，Escape 关闭并恢复编辑器焦点，所有开关都暴露按下状态。侧栏文件树后续应采用 tree/treeitem 语义和游标式焦点管理。
- 对比度与可读性：验证 13 px 弱化侧栏文本、选区、行号和禁用控件满足 AA 对比度；文字缩放后不得裁切。
- 屏幕阅读器：通过礼貌播报区通知文件、模式和保存状态变化；错误使用高优先级播报；暴露当前侧栏标签和大纲位置。
- 动效与感官：遵守 `prefers-reduced-motion`；状态同时使用文字、图标或形状表达；避免闪烁式保存反馈。

## 响应式行为

- 支持设备：macOS 桌面优先；当前 760 × 520 是窄桌面下限。Web 和 Windows 属于后续适配。
- 布局变化：宽度大于等于 960 px 时恢复记忆的侧栏宽度；760-959 px 默认收起侧栏，并以覆盖层打开；不能把正文画布压缩到不适合写作的宽度。
- 触摸与悬停：桌面控件可在悬停时显示上下文，但键盘聚焦时也必须出现；未来触摸界面使用至少 40 px 点击目标和明确点击操作。

## 交互状态

- 加载：安全情况下保留当前文档，只禁用冲突命令；在文档栏或提示区域标明当前操作。
- 空状态：区分无工作区、空文件夹、空文档和无标题大纲；每种状态只提供一个主操作和可选帮助。
- 错误：说明失败对象，保留用户内容，提供重试或修正操作，技术细节放入折叠区。
- 成功：保存结束后稳定显示“已保存”；复制和资源操作可使用短暂、非阻塞反馈。
- 禁用：解释操作不可用的原因，尤其是未保存文档粘贴图片或仅源码模式可用的操作。
- 离线或网络缓慢：核心写作和本地文件始终可用；未来 AI 功能必须独立显示离线/不可用，不能阻塞编辑。

## 文案风格

- 语气：简洁、平静、具体、以行动为导向。
- 术语：统一使用“文件”“文件夹”“大纲”“所见即所得”“源码”“已保存”“未保存”；原生菜单应跟随应用语言本地化。
- 微文案：指出失败对象和恢复操作，避免泛化的 `File operation failed`；正常界面不暴露 ProseMirror、runtime 或 raw fragment 等实现术语。

## 实现约束

- 框架与样式：React 19、Tauri 2、Milkdown/ProseMirror、CodeMirror 6、Tailwind 工具类和 `apps/desktop/src/styles.css`。
- 设计变量：优先扩展现有 `--theme-*`，不得另建第二套变量系统；修改相关界面时逐步替换硬编码颜色。
- 性能：输入更新保持局部；大纲和字数等派生计算需要防抖；保存状态或界面外壳更新不得重新挂载 Milkdown；只有测量证明必要时才虚拟化大型目录或文档。
- 兼容性：macOS 优先，但命令 ID 和交互语义必须可移植到 Web 和 Windows；Markdown/MDX 原始内容保真不可妥协。
- 测试与截图：为状态转换和键盘流程增加组件测试；为打开、编辑、模式切换、保存/放弃以及完整源码搜索键盘闭环增加集成测试；在 1120 × 760 和 760 × 520 下保留欢迎页、编辑、打开源码搜索、侧栏覆盖层、对话框和错误状态截图。最小窗口下，搜索面板不得遮挡当前匹配内容且必须完整可操作。

## 推荐实施顺序

1. 基础：文档栏、文件/窗口标题、保存/模式状态、真正收起侧栏、欢迎页和统一语言。
2. 连续性：光标/滚动/焦点恢复、资源返回导航、产品内保存/放弃/删除对话框和可操作反馈。
3. 写作能力：主题化源码搜索、选区工具栏、块插入菜单、链接/图片编辑、查找替换、命令面板和快捷键帮助。
4. 个性化：专注模式、打字机滚动、宽度/字体/主题设置，随后再将 AI 辅助作为独立且不阻塞写作的层加入。
